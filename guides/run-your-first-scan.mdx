## Automatic Scans

Once an asset is connected, CyBox scans automatically:

| Event | Scan Type |
|-------|-----------|
| Push to default branch | Full repo scan |
| Pull request opened or updated | PR diff scan (Firewall) |
| New URL asset added | DAST scan |
| Every 24 hours | Scheduled rescan |

You don't need to do anything — scans run automatically.

---

## Manual Scan

To trigger a scan on demand:

1. Go to **Assets** in the sidebar
2. Click on the asset you want to scan
3. Click **Scan Now** on the asset detail page
4. The scan is queued and will start within seconds

---

## Scan Progress

While a scan is running, the asset shows a **Scanning...** status. You can:
- Leave the page — the scan runs in the background
- Refresh **Findings** after a few minutes to see results
- Check the asset detail page for scan logs

---

## Scan Duration

| Asset Type | Typical Duration |
|-----------|----------------|
| Small repo (< 10k lines) | 30–60 seconds |
| Medium repo (10k–100k lines) | 1–3 minutes |
| Large repo (100k+ lines) | 3–8 minutes |
| URL / DAST scan | 2–5 minutes |

---

## What Happens If a Scan Fails?

If a scan fails (timeout, infrastructure error, or clone failure):
- The asset status shows **Scan Failed**
- Previous findings are not affected
- PR Firewall posts a neutral check (does not block the PR)
- You can retry the scan manually

For persistent failures, check the scan log on the asset detail page or contact [support@cybox.ai](mailto:support@cybox.ai).
