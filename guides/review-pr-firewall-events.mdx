# Review PR Firewall events

The PR Firewall view is where teams understand merge-time risk.

## What to review

- Verdict: Allow, Warn, or Block
- Which findings were introduced
- Severity mix
- Linked repository and pull request
- Whether the PR needs remediation before merge

## Good review habits

- Look at blocked PRs first
- Review repeated warning patterns across repositories
- Track override cases to learn where policy or tuning may need improvement
