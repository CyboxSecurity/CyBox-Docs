## When to Use AutoFix

Use AutoFix when:
- A finding has the **AutoFix Available** badge
- You want a fix PR generated without writing the code yourself
- You're addressing multiple SCA (dependency) findings at once

---

## Steps

<Steps>
  <Step title="Open a Finding">
    Go to **Findings** and click on any finding with the AutoFix badge.
  </Step>
  <Step title="Click AutoFix">
    Click the **AutoFix** button in the finding detail panel.
  </Step>
  <Step title="Wait for PR">
    CyBox generates the fix and opens a PR in your repository. This typically takes 15–30 seconds.
  </Step>
  <Step title="Review the PR in GitHub">
    Click the PR link from the finding detail. Review the change, check the description, and merge if it looks correct.
  </Step>
</Steps>

---

## What the PR Looks Like

Every AutoFix PR includes:
- **Branch**: `cybox/fix/{vuln-id}`
- **Title**: Clear description of the fix (e.g. "fix: upgrade lodash to 4.17.21 (CVE-2021-23337)")
- **Body**: The original finding details, what was changed, and any manual steps needed

---

## After Merging

Once the AutoFix PR is merged and the next scan runs:
- The finding status changes to **Fixed**
- The Security Risk Score updates accordingly
- The fix is tracked in the finding history

---

## Tips

<Tip>
For SCA findings, AutoFix upgrades the dependency to the minimum safe version — not the latest. This reduces the risk of breaking changes while resolving the CVE.
</Tip>

<Tip>
If AutoFix is not available for a finding, check the **Remediation Guidance** section in the finding detail — it provides specific steps to fix manually.
</Tip>
