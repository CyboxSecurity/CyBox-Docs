# FAQ

<AccordionGroup>
  <Accordion title="Who is CyBox for?">
    CyBox is built for security teams, engineering leaders, and fast-moving companies that want more control over AI-generated code and internet-facing software exposure.
  </Accordion>

  <Accordion title="Is CyBox only for AI-generated code?">
    No. CyBox is especially valuable in AI-assisted development environments, but it is also useful for broader software security workflows across repositories, pull requests, and public assets.
  </Accordion>

  <Accordion title="What does PR Firewall actually do?">
    PR Firewall evaluates the risk introduced by a pull request and returns a verdict so teams can decide whether to allow, review, or block the change before merge.
  </Accordion>

  <Accordion title="What is the difference between findings and posture?">
    Findings are individual issues. Posture is the broader view of how risk is trending across assets, teams, and time.
  </Accordion>

  <Accordion title="Does AutoFix replace developer review?">
    No. AutoFix helps teams move faster, but engineers should still review the proposed change before merging it.
  </Accordion>

  <Accordion title="Can CyBox help with public websites and external apps?">
    Yes. CyBox can track public-facing assets and help teams validate live exposure, not just source code risk.
  </Accordion>

  <Accordion title="Can we start small?">
    Yes. A great first rollout is one repository, one public asset, and one team. That is usually enough to prove value quickly.
  </Accordion>

  <Accordion title="How do I get help or request something?">
    Contact [support@cybox.ai](mailto:support@cybox.ai) for product help and [security@cybox.ai](mailto:security@cybox.ai) for security-related questions.
  </Accordion>
</AccordionGroup>
