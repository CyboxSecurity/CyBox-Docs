## Overview

Reports gives your organization a way to share security status with the people who need it — in the format that works for them. Go to **Reports** in the sidebar to access all available report types.

---

## Report types

### Security reports

<CardGroup cols={3}>
  <Card title="Executive Report" icon="file-lines">
    High-level security overview safe to share with customers and stakeholders. Summarizes posture, key findings, and trend.

    **External / Customer Safe**
  </Card>
  <Card title="Technical Audit" icon="shield">

    **Internal Only**
  </Card>
  <Card title="Compliance Evidence" icon="globe">

  </Card>

<CardGroup cols={1}>

---

## When to use each report

| Report | Best for |
|--------|----------|
| **Executive Report** | Board updates, customer security reviews, sales due diligence |
| **Technical Audit** | Engineering reviews, sprint planning, internal security assessments |
| **Compliance Evidence** | SOC 2, ISO 27001, and other audit frameworks |
| **Licenses & SBOM** | Supply chain transparency, procurement reviews, license compliance |

---

## Suggested rollout

Start with the **Executive Report** to establish a baseline view that leadership can read without context. Use **Technical Audit** exports for working sessions with engineering. Generate **Compliance Evidence** when preparing for an audit cycle.
