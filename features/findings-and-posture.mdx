## What Is a Finding?

A finding is a detected security issue in one of your assets. Each finding includes:

- **Title** — human-readable description of the issue
- **Severity** — Critical, High, Medium, Low, or Informational
- **Scanner** — which scanner detected it (SAST, SCA, Secrets, IaC, License, DAST)
- **Location** — file path and line number (for code findings) or URL and endpoint (for DAST findings)
- **Status** — Open, Fixed, or Ignored
- **Remediation guidance** — what to do about it
- **AutoFix availability** — whether a one-click fix PR is available

---

## Severity Levels

| Severity | Meaning |
|----------|---------|
| 🔴 **Critical** | Exploitable vulnerability with high impact. Fix immediately. |
| 🟠 **High** | Significant risk. Fix before next release. |
| 🟡 **Medium** | Moderate risk. Address in upcoming sprint. |
| 🔵 **Low** | Minor issue. Fix when convenient. |
| ⚪ **Informational** | No direct risk. Awareness only. |

---

## Finding Status

| Status | Meaning |
|--------|---------|
| **Open** | Active finding, not yet resolved |
| **Fixed** | No longer detected in the latest scan |
| **Ignored** | Manually dismissed (with reason) |

<Note>
CyBox uses fingerprint-based deduplication. The same vulnerability found in multiple scans is tracked as a single finding — not duplicated on every scan.
</Note>

---

## Security Posture Score

The **Security Risk Score** on your dashboard is a composite score across all open findings, weighted by severity:

- Critical × 15
- High × 5
- Medium × 1
- Low × 0.3

The score uses a sub-linear decay curve so a single critical finding has significant impact, but adding more low-severity findings does not linearly inflate the score. Lower is better.

---

## Filtering and Triage

From the **Findings** page you can filter by:

- **Severity** — focus on Critical/High first
- **Scanner** — isolate SAST, SCA, Secrets, etc.
- **Asset** — view findings for a specific repo or URL
- **Status** — Open, Fixed, Ignored
- **Introduced in PR** — view findings tied to a specific pull request

---

## Ignoring Findings

To ignore a finding:
1. Open the finding detail
2. Click **Ignore**
3. Select a reason (False Positive, Risk Accepted, Not Applicable)
4. Optionally add a note

Ignored findings are tracked with the reason and timestamp. They can be reinstated at any time.
