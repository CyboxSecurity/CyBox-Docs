# Live Exposure Validation

Source code tells part of the story. Live exposure tells you what an attacker can actually reach.

CyBox uses live testing on public web assets to help teams validate real external exposure.

## Why it matters

A clean repository does not always mean a safe deployment.

Real-world issues often come from:

- Misconfigured headers
- Exposed admin or debug paths
- Leaked files or documentation endpoints
- Unsafe redirects or cookie settings
- Unexpected internet-facing behavior after deployment

## What teams get from it

Live exposure validation helps teams:

- Confirm whether risk is truly reachable
- Focus remediation on assets that are publicly exposed
- Compare code-level findings with what is visible in production
- Investigate external-facing posture without leaving CyBox

## Good candidates for live testing

- Main production websites
- Customer-facing portals
- Preview or staging environments exposed to the internet
- Demo apps and pilot environments

## Suggested workflow

1. Add a website or domain
2. Run a live scan
3. Review findings and evidence
4. Fix issues or retire the asset
5. Re-scan to validate the improvement

<Warning>
Only test assets your team owns or is explicitly authorized to assess.
</Warning>
